ubuntu（客户端）配置shadowsocks

安装ss

sudo apt-get update
sudo apt-get install Python-gevent python-pip
pip install shadowsocks
配置ss
vim /etc/ss.json

以下内容为配置文件demo

{
"server":"服务器ip",
"server_port":服务器端口,
"local_port":1080,
"password":"密码",
"timeout":600,
"method":"aes-256-cfb"
}

启动ss

sslocal -c /etc/ss.json

设置开机启动

gnome-session-properties

在弹出的窗口里填写

添加(Add) -> 名称(name)和描述(comment)随便填

命令(Command)填写如下：

 nohup sslocal -c /etc/ss.json &

---------------------------------------------------------------------------------------------------------

配置polipo

Shadowsocks是我们常用的代理工具，它使用socks5协议，而终端很多工具目前只支持http和https等协议，对socks5协议支持不够好，所以我们为终端设置shadowsocks的思路就是将socks协议转换成http协议，然后为终端设置即可。仔细想想也算是适配器模式的一种现实应用吧。

想要进行转换，需要借助工具，这里我们采用比较知名的polipo来实现。polipo是一个轻量级的缓存web代理程序。

安装polipo

sudo apt-get install polipo

配置polipo

sudo vim /etc/polipo/config

配置文件如下

# Uncomment this if you want to use a parent SOCKS proxy:
socksParentProxy = "localhost:1080"
socksProxyType = socks5

重启polipo服务

sudo service polipo stop
sudo service polipo start

配置终端

bash中有一个很好的东西，就是别名alias。

修改~/.bashrc

vim ~/.bashrc

增加如下配置

alias wyl="http_proxy=http://localhost:8123"

刷新一下

source ~/.bashrc

测试一下

curl ip.gs
当前 IP：xxx.xxx.xxx.xxx 来自：中国安徽马鞍山 电信
wyl curl ip.gs
当前 IP：xxx.xxx.xxx.xxx 来自：日本东京

git配置代理

修改~/.bashrc

vim ~/.bashrc

增加如下配置

sw=" --config http.proxy=localhost:8123"

刷新一下

source ~/.bashrc
